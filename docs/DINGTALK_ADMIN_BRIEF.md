# 美妆法务资讯机器人上线准备说明

这份说明是发给钉钉企业管理员看的。管理员只需要按清单准备信息，不需要了解程序代码。

## 这个机器人做什么

我们准备上线一个“美妆法务资讯机器人”。

它会定期生成美妆行业法务资讯周报，然后：

1. 自动写入钉钉知识库文档；
2. 再往指定钉钉群发送一条摘要消息；
3. 群成员点击摘要里的链接，可以查看完整周报。

## 需要管理员帮忙做什么

### 1. 创建或指定一个企业内部应用

请在正式钉钉企业里创建一个企业内部应用。

应用名称建议：

```text
美妆法务资讯机器人
```

创建后，请提供：

```text
企业名称：
应用名称：
APPID：
Client ID：
Client Secret：
```

说明：`Client Secret` 是密钥，请不要发到公开群里。

### 2. 给应用开通知识库和文档权限

请给这个应用开通以下权限：

```text
Document.WorkspaceDocument.Read
Document.WorkspaceDocument.Write
Wiki.Workspace.Read
Wiki.Node.Read
Document.Workspace.Write
Wiki.Workspace.Search
```

如果后续希望机器人自动创建新的知识库，也请额外开通：

```text
Wiki.Workspace.Write
```

简单理解：

- 读取知识库信息；
- 在知识库里创建文档；
- 写入和更新周报文档；
- 查找已有文档，避免重复。

### 3. 发布应用并完成授权

权限添加后，请确认：

```text
应用是否已发布：是 / 否
权限是否已授权：是 / 否
```

如果应用没有发布，或者权限没有授权，机器人将无法写入知识库。

### 4. 准备一个正式知识库

请创建或指定一个正式知识库，例如：

```text
美妆法务资讯周报
```

请提供：

```text
知识库名称：
知识库链接：
知识库 Workspace ID：
```

如果暂时不知道 Workspace ID，可以先提供知识库链接，后续由项目对接人协助整理。

### 5. 指定一个操作人账号

机器人写文档时，需要指定一个“操作人账号”。

建议使用稳定账号，例如：

- 法务负责人账号；
- 企业管理员账号；
- 公司系统账号。

不建议使用临时员工账号，避免后续人员变化影响机器人运行。

请提供：

```text
操作人姓名：
操作人 User ID / operatorId：
操作人是否能编辑该知识库：是 / 否
```

### 6. 创建正式钉钉群机器人

请在正式接收周报的钉钉群里创建一个自定义机器人。

机器人名称建议：

```text
美妆法务资讯机器人
```

安全设置建议：

```text
关键词：美妆法务资讯
加签：开启
```

请提供：

```text
钉钉群名称：
机器人 Webhook 地址：
机器人关键词：
是否开启加签：是 / 否
加签 Secret：
```

说明：正式环境建议开启“加签”，这样更安全。

## 后续怎么处理

管理员把以上信息提供给项目对接人后，项目对接人会整理配置。

正式切换前，会先做一次测试：

1. 测试是否能创建知识库文档；
2. 测试是否能写入周报内容；
3. 测试是否能向钉钉群发送摘要消息。

确认无误后，再切换为正式每周自动推送。
